## 🎯 사용 방법

### Step 1: 개발자에게 문서 전달
```bash
# 백엔드 개발자에게
docs/pm/handoff/2026-02-09_구현상태확인_백엔드.md

# 모바일 개발자에게
docs/pm/handoff/2026-02-09_구현상태확인_모바일.md
```

### Step 2: 작성 가이드 전달
각 개발자에게 다음을 안내:

1. **체크박스 체크**
   - ✅ 완료
   - 🚧 부분 완료
   - ❌ 미완료
   - ⚠️ 이슈 있음

2. **완료율 입력**
   - 0% ~ 100%

3. **테스트 여부**
   - Yes / No

4. **빈 칸 채우기**
   - `_______` 부분에 정보 입력

5. **알려진 이슈 상세 작성**
   - 재현 방법
   - 영향 범위
   - 해결 방안 (있다면)

### Step 3: 회수 및 검토
- **기한**: 2026-02-10 (1일 이내)
- **검토**: PM이 회수 후 검토
- **후속 조치**: 필요 시 추가 질문

### Step 4: 문서 최신화
- 로드맵 업데이트
- 구현 상태 보고서 업데이트
- 이슈 트래킹 시스템에 등록
